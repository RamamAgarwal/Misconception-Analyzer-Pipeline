# Misconception Analyzer Pipeline

A lightweight Python data pipeline that processes student interaction logs, uses an LLM to diagnose specific misconceptions, and produces a per-student teacher report with time-weighted mastery scores.

---

## Project Structure

```
├── student_logs.json    # Input: 10 valid + 2 intentionally malformed records
├── pipeline.py          # The entire pipeline (load → analyze → aggregate → report)
├── teacher_report.json  # Output: generated by running pipeline.py
└── README.md
```

---

## Setup or Usage

```bash
pip install openai pandas
export OPENAI_API_KEY="your-key-here"
python pipeline.py
```

> **OpenAI client:** The `API_KEY` and `MODEL` constants are at the top of `pipeline.py`. This project uses the OpenAI client (`openai`) and expects `OPENAI_API_KEY`.

## Design Decisions

### I. Data Processing / Data Validation

Utilises a function *load_and_clean* which  load JSON logs into a validated Pandas DataFrame, drops rows that are missing fields, have wrong types, or bad timestamps and finally logs a warning for every skipped row so the issue is traceable.

Validation is layered, not monolithic where each check is a distinct step with its own log warning, making it easy to trace exactly why a row was dropped. Two malformed rows are included to test the pipeline:

| Row    | Flaw                                     | Handling                                      |
| ------ | ---------------------------------------- | --------------------------------------------- |
| Row 10 | `is_correct: "yes"` (string, not bool) | Caught by type check, skipped                 |
| Row 11 | `timestamp: "not-a-real-date"`         | Caught by `datetime.fromisoformat`, skipped |

### II. LLM Misconception Analysis

Utilises a function *call_llm* which calls the LLM and parse a structured JSON response, Retries on timeout or JSON parse failure with exponential back-off and finally returns a safe fallback dict on total failure.

Two strategies are implemented and can be switched via `STRATEGY = "zero_shot" | "chain"`:

**Zero-Shot** : Sends all facts to the LLM at once in one prompt and ask it to infer the misconception directly. Simple, fast and sufficient for arithmetic errors where the mistake is self-evident from the numbers alone.

**Chain (Prompt Chain / CoT)** : Uses a two-step approach:

1. Ask the model to solve the problem step-by-step (establishes ground truth reasoning).
2. Given that solution, ask it to pinpoint exactly where the student diverged.

**Which worked better?** The chain approach consistently produced more specific diagnoses because the model reasons about the correct path before comparing it to the student's path, reducing hallucinated misconceptions. For example, on `s = ut + 0.5*a*t²` with `t=3`, zero-shot returned *"incorrect formula application"* while the chain returned *"student multiplied a×t but forgot the 0.5 factor and used t instead of t²"* - the latter is directly actionable for a teacher. The one trade-off is latency (~2 API calls vs 1), which is acceptable in a batch pipeline.

The system prompt enforces strict JSON output to prevent hallucinations and free-form text, with a 3-retry loop handling timeouts and parse failures.

### III. Concept Mastery Score

The score is **not** a simple accuracy percentage. It uses two layered signals:

**Base score (time-weighted):**

```
weight_i = e^(rank / total_attempts)    # recent attempts weighted higher
score = Σ(weight_i × correct_i) / Σ(weight_i) × 100
```

This score rewards improvement over time, a student who got the first 3 wrong but nailed the last 2 scores higher than one who got the same 2 right but did them first. This is far more useful to a teacher than a flat percentage.

**Severity deduction (LLM-informed):**

```
final_score = max(0, base_score − Σ severity_deductions)
```

| Severity | Deduction | Rationale                                            |
| -------- | --------- | ---------------------------------------------------- |
| low      | −5 pts   | Surface error (e.g., sign flip), quick fix          |
| medium   | −12 pts  | Procedural gap, needs targeted practice             |
| high     | −22 pts  | Deep conceptual misunderstanding, needs re-teaching |

A student can have a high accuracy rate but a low mastery score if their one wrong answer reveals a fundamental misconception which is exactly the kind of signal a teacher needs.
